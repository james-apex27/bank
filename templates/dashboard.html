About Bank of Apex27
What is Bank of Apex27?

Bank of Apex27 is a self-contained bank simulator built specifically for testing Apex27's bank file import and export features. Rather than connecting to a real bank during development and testing, you can create fictional bank accounts here, populate them with transactions that mimic real-world activity — rent payments, holding deposits, security deposits, refunds — and generate the exact file formats that Apex27 expects to receive.

Each account is assigned a randomly generated sort code and account number and is tied to one of three supported bank formats: Barclays, NatWest, or Sage. The bank type determines the CSV structure used when exporting a statement, ensuring the file Apex27 imports matches the format it has been configured to handle for that bank. Transactions can be added manually with a date, description, amount, reference, and type — giving you full control over the test data without needing a live banking connection.

The export system tracks which transactions have already been sent to Apex27. Each time you download a statement, only the new, un-exported transactions are included, and they are automatically marked as exported afterwards. This mirrors how a real bank feed works, allowing you to test incremental imports rather than re-importing the same data every time. Individual transactions can be reset if you need to re-run a specific test.

The simulator also supports the reverse flow. When Apex27 generates a BACS payment file — for example, to pay out to landlords — you can upload that file here via the Import BACS page. The simulator will match each payment row to the corresponding account by sort code and account number and apply the transaction automatically. Any rows that do not match an existing account are shown separately so you can identify gaps in your test setup. Together, these two flows let you verify the complete round-trip: money coming in to Apex27 via statement import, and money going out via BACS export.
